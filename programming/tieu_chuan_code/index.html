    Tiêu chuẩn về cách đặt tên#

    Class (lớp)#

    • Tên 1 class phải thể hiện được ý nghĩa ngắn gọn của class đó, tên class sử dụng chữ cái, chữ cái đầu tiên phải viết hoa. ví dụ như chức năng quản lý Menu thì sẽ cần những class sau :
    MenuControler: Class xử lý controler

MenuModel: Class xử lý vềModel

MenuHelper: Class hỗ trợ xử lý các nghiệp vụ phức tạp

    Method (phương thức)#

    • Tên 1 phương thức sẽ thể hiện một hành hành động cụ thể của class, phương thức sử dụng chữ cái thường để biểu diễn. ví dụ trong chức năng quản lý Menu sẽ cần có những phương thức sau:
    index: Khởi tạo dữ liệu mẫu, load các file js,css liên quan

loadlist: Load màn hình danh sách các menu

add: Thêm mới một menu

edit: Sửa một menu

delete: Xóa một menu

update: Cập nhật một menu

    Properties (Thuộc tính)#

    • Hằng số: Hằng số sử dụng chữ cái viết hoa để thể hiện. vd: const VERSION = '1.0';
    • Biến: sử dụng chữ cái viết thường để thể hiện, gắn giá trị cho biết sử dụng ký tự ngoặc đơn '', nếu biến lưu giá trị của một array hoặc 1 object thì thêm chữ 's' ở cuối, biến lưu giá trị 1 string thì không. ví dụ: 
    user = 'user1,uer2' // biến lưu giá trị chuỗi viết thường

users = array( // biến lưu giá trị mảng hoặc object thêm chữ s ở cuối

                 0 => array('name'=>'user1'),

                1 => array('name'=>'user2')

);

    Table (bảng):#

    • Tên bảng phải ngắn gọn thể hiện được ý nghĩa của bảng đó, sử dụng chữ cái viết thường và ký tự "_" để đặt tên cho bảng
    • Đặt tên theo quy tắc: [module-prefix]_[ten-bang]. (lưu ý: chỉ đặt module-prefix cho những bảng thuộc các loại sau: bảng liên quan đến hệ thống, bảng liên quan đến dịch vụ ivan, ihoadon, cks) Một số ví dụ

    ví dụ:

    user : bảng người dùng

company : bảng lưu thông tin các công ty

system_list: bảng lưu danh mục (system là [module-prefix]; list là tên bảng) 

ivan_customer: bảng lưu khách hàng trong dịch vụ ivan ( ivan là [module-prefix]; customer là tên bảng) 

Lưu ý: đặt tên bảng sẽ khởi tạo trong migration của laravel nên nó sẽ cộng với db_prefix mình khai  báo do đó có thể trong database tên bảng sẽ khác như: t_user, abc_user, xyz_user, abcxyz_user tùy thuộc vào người quản trị đặt db-prefix

    Column (cột)#

    • Tên cột phải thể hiện được các thuộc tính của đổi tượng bảng, sử dụng chữ cái viết thường và ký tự "_" để đặt tên cho cột.
    • Đối với khóa chính (primary key) của bảng mặc định phải đặt là id
    • Đối với khóa ngoại (foreign key) thì đặt theo quy tắc sau: [ten-bang-lien-ket]_id Ví dụ về các cột trong bảng user:
    id                                     // khóa chính của bảng mặc định đặt là id

company_id                // là khóa ngoại của bảng company

name                             // Tên người dung

age                                 // Tuổi người dùng

sex                                // Giới tính

    Storeprocedure:#

    Tên store đặt theo quy tắc: [ten_module]_[ten_controller][hanh_dong]. ví dụ:

    Category_ConfigGetAll (như vậy strore thuộc module: Category, thuộc controller: Config với hành động lấy tất cả chuyên mục getall)

    If else#

    1- Lệnh if 1 điều kiện

if ($a != 2) {

      $a = 2;

}

2- Lệnh if else

if ($a != 2) {
    $a = 2;
} else {
    $a = 7;
}

3- Lệnh if else if

if ($a != 2) {
    $a = 2;
} elseif ($a == 3) {
    $a = 4;
} else {
    $a = 7;
}
